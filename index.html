Git e GitHub

git init nome-do-arquivo

Indica qual arquivo começará a ser versionado pela ferramenta git.

git config --global user.name "nome-de-usuario"

Informa o nome de usuário que realizará as mudanças no arquivo.

git config --global user.email "email-do-usuario"

Informa o nome de email do usuário que realizará as mudanças no arquivo.

git status

Mostra o status do(s) arquivo(s), que precisa(m) ser adicionado(s) para ocorrer a mudanças.

git add nome-do-arquivo

Adiciona o arquivo ao repositório.

git commit -m "descrição da atualização"

Após ter adicionado o arquivo, descrever as mudanças que foram realizadas.

git log

Registro de mudanças realizado no arquivo.

git log --all

Mostra o log de todos os branch's.

git show

Mostra o que foi alterado na última atualização.

git diff hash(1) hash(2)

Mostra as mudanças acrescentadas/excluídas entre os dois momentos.

git blame nome-do-arquivo

Mostra quem realizou as mudanças no determinado arquivo.

git branch nome-da-branch

Cria um novo "canal" para realizar as mudanças.

git checkout nome-da-branch

Escolho a branch que desejo trabalhar.

git merge nome-da-branch

Unifica os dados com a branch citada.

ssh-keygen -t rsa -b 4096 -C "email-do-usuario"

Cria um par de chaves assimétrico.

eval "$(ssh-agent -s)"

Iniciar um agente de comunicação com o servidor.

ssh-add ~/.ssh/id_rsa

Adiciona o agente.

Inserindo as informações no git

Após toda a configuração, o usuário deverá cadastrar seu id_rsa.pub em settings -> SSH and GPG Keys.

git remote add origin nome-do-repositório

Pega as informações do repositório remoto.

git clone nome-do-repositório

Realiza uma cópia do repositório remoto. Opção utilizada quando o projeto esta em andamento.

git push

Envia as informações para o repositório remoto.

git config --global alias.nome-da-variavel "instrução do codigo"

Criando variaveis ou apelidos para simplificar rotinas.

git tag -a v0.0.0 (insira a versão desejada) hash

Cria a versão do seu código.

git tag

Exibe todas as tags.

git show-ref --tags

Ver o commit que esta associado aquela tag.

git tag -d v0.0.0 (insira a versão desejada)

Apaga a versão da Tag.

git push --tags

Envia as tags para o servidor do GitHub.

Sobre o .gitignore

Cria-se um arquivo, cujo o nome é .gitignore e dentro o usuário deve informar quais arquivos deve ser ignorados.

Sobre o FORK

O usuário recebe os arquivos do proprietário em seu repositório, podendo atualizá-los sem afetar o proprietário.

Sobre Pull Request

É uma maneira de verificar o código junto ao seu colegas antes de realizar o git merge.
